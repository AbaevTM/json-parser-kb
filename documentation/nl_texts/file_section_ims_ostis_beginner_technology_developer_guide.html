Для того, чтобы быстро войти в курс дела по разработке Метасистемы IMS.OSTIS рекомендуется выполнить следующее:
  1. Познакомиться с общей структурой Проекта OSTIS, направленного на создание и распространение Технологии OSTIS, путём навигации по спецификации Проекта OSTIS и его подпроектов.
  2. Детально познакомиться со структурой Проекта IMS.OSTIS, направленного на разработку Метасистемы IMS.OSTIS и являющегося основным (!) подпроектом Проекта IMS.OSTIS
  3. Просмотреть структуру (оглавление) Документации Технологии OSTIS, разделы которой представляют собой не что иное, как результаты (продукты) всевозможных подпроектов Проекта IMS.OSTIS. Иерархия разделов указанной документации полностью соответствует иерархии проектов, входящих в состав Проекта IMS.OSTIS.
  4. При изучении Документации Технологии OSTIS особое внимание уделить изучению Раздела IMS.OSTIS, в котором описана архитектура Метасистемы IMS.OSTIS и содержится полная документация (полное описание) этой метасистемы как на уровне её логико-семантической модели (sc-модели), так и на уровне технической реализации этой модели.
  5. Выбрать интересующий Вас атомарный подпроект Проекта IMS.OSTIS, то есть такой проект, входящий в состав Проекта IMS.OSTIS, который не имеет подпроектов. По спецификации выбранного проекта изучить (1)состав его исполнителей и их роли (обязанности) в рамках этого проекта, (2)текущее состояние проекта и, в частности, его проектные задачи: предлагаемые проектные задачи (задачи-черновики), которые могут быть предложены кем угодно; утверждаемые проектные задачи – задачи, находящиеся на согласовании (задачи, необходимость выполнения которых оценивается руководителем соответствующего проекта); горячие проектные задачи – задачи, необходимость выполнения которых одобрена, но исполнители пока не определены; исполняемые проектные задачи; проверяемые проектные задачи – задачи, результаты выполнения которых в текущий момент проверяются (тестируются, рецензируются) завершённые проектные задачи – задачи, результаты выполнения которых прошли проверку успешно. В случае, если указанная проверка прошла неуспешно (выявлены ошибки), создаются новая исполняемая проектная задача, направленная на исполнение выявленных ошибок, исполнителем которой назначается тот, кто эти ошибки совершил.
  6. В рамках выбранного атомарного подпроекта Проекта IMS.OSTIS либо создать новую предлагаемую проектную задачу, предложив себя в качестве будущего её исполнителя, либо выбрать одну из горячих проектных задач, предложив себя в качестве её исполнителя. Указанные предложения направляются по электронной почте руководителю соответствующего проекта.
